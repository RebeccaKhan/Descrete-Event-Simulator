5.3 Analysis and General Questions¶

Task 5.3.1: Confidence Interval Width¶

Firstly we observe that we need more runs when we decrease alpha, which is an expected result as we need to more values to fall closer to each other and as we have established in the previous assignments, the variance of the blocking probability decreases over time. Having lower aplha is like having a stricter limit on the variace of the blocking probabilities and hence we see that it takes more time/runs. We see that the overall time needed to reach a certain confidence interval is more or less the same even with smaller batches or smaller simulation time (for the same alpha). However, for part 5.2.2 we always require more time than 5.2.1 because in 5.2.1 we start each run with an empty buffer(transient phase) and the blocking probability of the initial packets will be less. This same reason can be applied to explain why the time required to reach the confidence interval is more for higher simulation time or higher batch sizes. The disadvantage of the method in Task 5.2.1 is that we have a transient phase in each simulation run. The variance in the blocking probability is higher in the transient phase hence it is not optimum as explained above. One solution can be to start with a filled buffer. The disadvantage of the method in Task 5.2.2 is that for higher RHO the blocking probabilities of consecutive packets get more correlated and hence we will not have independent runs. A solution to this can be to have "guard" patches between sample runs to make sure that the consecutive packets are uncorrelated.

Task 5.3.2: Confidence Interval Width¶

The theoretical blocking probability for S=4 and RHO=0.9 is 0.126. I printed the mean blocking probability at the end of each test in each Task, we see that for a higher simulation time or higher batch size the calculated blocking probability is closer to the theoretical one. This is because the analytical formula is for a steady state which reaches after a long simulation time and hence we must siulate for longer time ( ideally infinite) to reach the exact steady state blocking probability.

Task 5.3.3: Confidence Interval Width¶

From the plots (notice the Y-axis as its not the same scale for all the plots), we see that the width of the confidence interval is larger for smaller simulation time, which is expected because the variance of utilization is higher during the transient phase. For a smaller alpha, less number of values cover the true value. This is also expected as we are essentially reducing our tolerance for variance of the same test. We see that running mean of the sample converges to the theoretical RHO value ( since we assume an infinite Buffer size) as the number of runs increases. If we simulate of infinite number of rus, we will achieve the theoretical RHO. We see the effect of skewness when we increase the RHO where, very few samples cover the theoretical RHO and that is due the fact that we have many blocked packets in the buffer which are xet to be processed ( because of the infinite buffer)

Task 5.3.4: Variable Simulation Time¶

It would be possible to check the BP, each time a packet is droped and see if it reaches inside the confidence interval, this will give us more granularity and our result will not have to be in multiples of 100 or 1000 seconds or 100 or 1000 runs. We will first have to ignore the first few values as the blocking probaiblity of the packets in the startup phase will anyways be lower. The other problem with this mehtod will be that blocking probabilities of consecutive packets are correlated and hence we would not have independent runs.

Task 5.3.5: Bootstrapping¶

Bootstrapping is useful when we have a finite number of samples which we know re not out entire population size. In case of regulat confidence intervals, we assume that our sample is the entire population size and has according to the central limit theorem a normal or student-t distribution. However, if this is not the case we are introducing bias in our estimate. With bootstrapping we can reduce the effect of this bias. The bootstrap and regualr confidence intervals will be the same in case we take infinite bootstrap samples.
